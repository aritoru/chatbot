## ADDED Requirements

### Requirement: Agent system prompt SHALL reflect current Frustration Signal

The system prompt generated by `_build_system_prompt` SHALL include a Frustration Signal directive that instructs Claude how to modulate its conversational style. The directive SHALL map signal levels to distinct behavioral guidance.

#### Scenario: Frustration Signal is None

- **WHEN** `Session.frustration_signal` is `None`
- **THEN** the system prompt contains no frustration directive and the Agent behaves normally

#### Scenario: Frustration Signal is Mild

- **WHEN** `Session.frustration_signal` is `Mild`
- **THEN** the system prompt includes a directive instructing the Agent to acknowledge any difficulty, reduce the number of clarifying questions in the current turn to at most one, and use a warmer, more empathetic tone

#### Scenario: Frustration Signal is High

- **WHEN** `Session.frustration_signal` is `High`
- **THEN** the system prompt includes a directive instructing the Agent to skip any optional clarifying questions, infer remaining unknown fields from context with reduced evidence threshold, and move toward Intake confirmation as quickly as possible

### Requirement: Agent SHALL NOT mention frustration to the Customer

The Agent SHALL NOT directly tell the Customer that frustration has been detected, nor reference any internal Frustration Signal classification in the Customer-facing turn.

#### Scenario: High Frustration Signal turn — no disclosure

- **WHEN** `Session.frustration_signal` is `High`
- **THEN** the Agent's response does NOT contain phrases such as "I can tell you're frustrated", "I notice you seem annoyed", or any equivalent disclosure of the internal classification

### Requirement: Agent SHALL skip optional clarifying questions when Frustration Signal is High

When `Session.frustration_signal` is `High`, the Agent SHALL proceed toward confirmation using only the information already provided, filling in any missing constrained fields via inference. The Agent SHALL NOT ask more than one question in a `High`-signal turn, and only if a required field cannot be inferred at all.

#### Scenario: All fields inferable — skip clarifying questions

- **WHEN** `Session.frustration_signal` is `High`
- **AND** all four Intake fields can be inferred from the transcript
- **THEN** the Agent moves directly to the Confirmation Step without asking any further questions

#### Scenario: One field still missing — single targeted question

- **WHEN** `Session.frustration_signal` is `High`
- **AND** exactly one required field has no inferable value
- **THEN** the Agent asks one concise question to collect that field and does not ask anything else in the same turn
